[✅ DONE] Dead-ball loop (игроки исчезают с поля)
  Три итерации исправления, итоговое решение:
  — MovementSystem игнорирует nextDecision.target во время dead-ball фаз
  — triggerRestart() сбрасывает nextDecision = null у ВСЕХ игроков
  — doKickoff() то же самое перед каждым кикоффом
  — DecisionSystem / OffBallSystem / PassingSystem / ShootingSystem /
    TackleSystem полностью пропускают работу в dead-ball фазах
  — lockTakerToRestartPos() каждый тик держит тейкера на месте рестарта
  — Таймаут 3 сек: принудительный resume если фаза застряла

[✅ DONE] Tackle спам
  TACKLE_SUCCESS_COOLDOWN = 150 тиков. Guard через actionCooldown.

[✅ DONE] Goalkick бьёт вратарь (раньше был excluded из eligible)

[✅ DONE] Halftime / Goal delay + корректный kickoff
  resetFormationPositions + очистка nextDecision перед возобновлением

[✅ DONE] 1.1 Command System
  Registry-based handlers, типизированные через mapped types.
  resolver.register() для расширения без правки core.
  UPDATE_PLAYER_METRICS — движок без прямых мутаций.

[ ] 1.2 Immutable Safety
  readonly snapshots, deep freeze в dev режиме,
  mutation assertions для поиска скрытых side effects.
  → Важно: сейчас RefereeSystem всё ещё мутирует player.pos напрямую
    (taker.pos, taker.targetPos и т.д.) — это нарушает паттерн команд.
    Идеально перевести на команды TELEPORT_PLAYER / SET_PLAYER_TARGET.

[ ] 1.3 Tick Pipeline (формализовать порядок)
  Сейчас порядок систем зашит в MatchSimulator.
  Нужно: декларативный граф зависимостей с явными read/write контрактами.
  Это позволит поймать конфликты типа "MovementSystem читает то,
  что RefereeSystem ещё не записал".

[ ] 1.4 Event Bus 2 уровня
  Simulation Events (для логики): possession_changed, interception
  Presentation Events (для UI): crowd_reaction, commentary, highlight

[✅ DONE] 2.4 Tactical States (TeamTacticalState)
  5 фаз: in_possession / transition_attack / out_of_possession /
  transition_defend / set_piece. Обновляются каждый тик.

[✅ DONE] 2.5 Decision Scoring (UtilityAI)
  HoldAction, PassAction с lane bonus, DribbleAction, ShootAction.

[🔧 PARTIAL] 2.1 Off-ball Intelligence
  OffBallSystem существует: support_run, overlap_run, underlap_run,
  third_man_run, defensive_recovery, hold_shape.
  НО: во время dead-ball система отключена, и после рестарта
  игрокам нужно несколько секунд чтобы получить новые решения.
  Нужно: явный "repositioning phase" после рестарта где
  каждый игрок получает цель — свою formation позицию.

[🔧 PARTIAL] 2.2 Space Awareness
  SpaceAwareness.compute() есть, используется в OffBallSystem.
  Нужно: pressure zones, dangerous zones (зоны перед воротами),
  учёт defensive line при выборе передачи.

[🔧 PARTIAL] 2.3 Team Shape System
  Centroids и compactness есть. Нет: dynamic width/depth adjustment
  в зависимости от фазы и счёта.

[ ] 2.6 Restart Intelligence (НОВОЕ, приоритет)
  Сейчас после throwin/goalkick/corner игроки просто бегут на
  formation позицию. Нужно:
  — при corner: игроки занимают зоны в штрафной
  — при throwin: ближайшие создают варианты для передачи
  — при goalkick: команда перестраивается для получения
  — при freekick: стенка + варианты удара

Самая опасная ошибка футбольных движков:
улучшать renderer / анимации вместо принятия решений.

Качество игры определяется:
positioning / spacing / decision timing / tactical logic
— а не красотой рендера.

Текущий architectural core хороший.
Следующий барьер — restart intelligence и error model.
Без ошибок мяч никогда не уходит за линию "случайно",
без умных рестартов каждый аут выглядит как баг.
